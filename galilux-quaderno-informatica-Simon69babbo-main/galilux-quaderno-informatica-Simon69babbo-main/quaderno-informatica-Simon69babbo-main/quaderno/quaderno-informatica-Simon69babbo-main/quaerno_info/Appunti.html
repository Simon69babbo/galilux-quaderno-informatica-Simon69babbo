MODELLO E/R

IL MODELLO E/R e' un modello concettuale utilizzato per la progettazione di basi di dati. Le sue caratteristiche essenziali sono:

ENTITA' : Rappresentano gli oggetti o concetti reali o astratti che hanno una certa rilevanza per il sistema. Ogni entita' e' caratterizzata da un insieme di attributi che ne definiscono le proprieta'. Ad esempio, in un database di una biblioteca, un'entita' potrebbe essere "Libro", e gli attributi potrebbero essere "Titolo", "Autore", "Anno di pubblicazione", ecc.

RELAZIONI : Definiscono come le entita' interagiscono o sono collegate tra di loro. Una relazione puo coinvolgere una o piu' entita'. Ad esempio, la relazione "Prestito" tra l'entita' "Utente" e l'entita' "Libro" in una biblioteca.

CARDINIALITA': La cardinalita' di una relazione indica quante entita' di un tipo possono essere associate ad entita' di un altro tipo. Le cardinalita' piu' comuni sono:
Uno a uno (1:1): Ogni istanza di una entita' e' associata a una sola istanza di un'altra entita'.
Uno a molti (1:N): Un'istanza di una entita' e' associata a molte istanze di un'altra entita'.
Molti a molti (N:M): Molte istanze di una entita' possono essere associate a molte istanze di un'altra entita'.

CHIAVE PRIMARIA: Ogni entita' e' identificata in modo univoco tramite un attributo o un insieme di attributi che fungono da chiave primaria. Questo e' fondamentale per garantire l'integrita' e l'individualita' dei dati.

DATABASE

Il comando CREATE TABLE in SQL viene utilizzato per creare una nuova tabella in un database. Quando esegui questo comando, definisci la struttura della tabella, comprese le colonne (campi) e i tipi di dati che ogni colonna puo' contenere.

CREAZIONE DEL CODICE (CREATE TABLE) IN SQL

CREATE TABLE nome_tabella (
nome_colonna1 tipo_dato1 [VINCOLO1],
nome_colonna2 tipo_dato2 [VINCOLO2],
...
nome_colonnaN tipo_datoN [VINCOLON]
);

SPIEGAZIONE DEI COMPONENTI:

nome_tabella: e' il nome che vuoi assegnare alla nuova tabella.

nome_colonnaX: e' il nome di ciascuna colonna della tabella.

tipo_datoX: e' il tipo di dato che la colonna puo' contenere: (ad esempio, INT, VARCHAR, ecc.).

VINCOLO (opzionale): e' un vincolo che definisce regole aggiuntive per le colonne come: NOT NULL, PRIMARY KEY, UNIQUE, DEFAULT, FOREIGN KEY, ecc.

QUERY

Una query e' una richiesta di informazioni a un database. In altre parole, e' un comando che viene inviato al sistema di gestione del database (MySQL) per ottenere, inserire, aggiornare o eliminare dati.

AD ESEMPIO:
una query di selezione (SELECT) viene utilizzata per ottenere dei dati specifici da una o più tabelle del database.
Una query può anche essere usata per manipolare i dati, come inserire nuove righe (INSERT),
aggiornare dati esistenti (UPDATE),
o eliminare righe (DELETE).

Esempio di una query di selezione:
SELECT Nome,
Cognome FROM Cliente,
WHERE Codice_Cliente = 1;

Questa query restituisce il Nome e il Cognome di un cliente con Codice_Cliente uguale a 1. Le query sono fondamentali per interagire con il database e ottenere i risultati desiderati.
